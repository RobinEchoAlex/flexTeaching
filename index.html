What is flexTeaching?

flexTeaching is a teaching tool for creating and deploying lessons and assignments that include personalized random data, based on a random seed based on their student ID. It is based on Rstudioâ€™s shiny and flexsdashboard.

See it in action

Go to the solutions demo, choose the "summaries" assignment, and uncheck "Hide answers?". Notice that the solutions appear within the document. Now change the student ID or the secret, and see how the data set, and the solutions, change. If you change the student ID or secret back to what it was, you will see that the data and solutions are the same as they were.

How does it work?

A student goes to the data download interface (demo) and selects an assignment. They then enter their unique ID (e.g., student ID). They can then read the assignment and download a randomly-generated data set that is specific to them. The data is generated using a secret that the student does not know, but markers do.

One the student has turned in their assignment, a marker can then go to the solution interface (demo) and input the student's ID number, along with the secret that was used to generate the data. If they uncheck "Hide answers?" they can view the assignment along with the solutions specific to that student.

The students can also access the solutions interface (demo), but because they don't know the secret they do not get the solutions to their data set. But they can download a new data set and see the solutions for that data set. By changing the ID and the secret in the solutions interface, they can obtain as many practice data sets as they like.

How can I try making my own assignment?

Making an new assignment is easy.

1. Get the repository

If you know how to use GitHub, fork the repository. If not, download the repository and unzip the file.

2. Create a new assignment

Duplicate the folder app/assignments/summaries in the same folder; call it anything else, say, "summaries2".

3. Install the necessary packages

Run this line of code in R/RStudio:

 install.packages(c("shiny","rmarkdown","stargazer","broom","xtable",
 "flexdashboard","digest","base64enc","haven","xlsx",
 "lubridate"), dep=TRUE)

4. Start the flexTeaching app

Open solve.Rmd in RStudio and click "Knit" or "Run document" (depending on what it says in your interface).

You should see your "new" assignment listed in the assignment box. Selecting it should give you the same output as the summaries" assignment.

5. Edit the data generating function

Open assignments.R in your new assignment's folder, and edit the getData function so that it returns a data.frame with random. Do not change the set.seed.alpha line.

6. Edit the assignment markdown

Open assignments.Rmd file. On code chunks that should only be output when solutions are requested, use the arguments eval=solutions or include=solutions. The former does not run the chunk, the latter runs it but does not make the results visible.

7. Restart the interface

Open solve.Rmd in RStudio and click "Knit" or "Run document". You should be able to see the changes you made to your assignment. Uncheck the "Hide answers?" box to see the solutions.

8. Publish on a shiny server

To make your assignments visible and usable by others, you must publish them on a shiny server such as (shinyapps.io) or one set up by your institution.
